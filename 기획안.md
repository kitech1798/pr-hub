# 기관 홍보 허브 — 기획안

작성일: 2026-05-07 (2026-05-07 1차 갱신: 카테고리 4종 확정, 인스타 제외, v1 = 단순 리스트)
담당: 정재혁 (홍보실)

---

## 1. 비즈니스 문제

현재 우리 원의 홍보 콘텐츠는 한곳에 모인 허브가 전무하다.

- **보도자료**: 담당자 개인 파일 또는 홈페이지 게재 내역을 직접 찾아야 함
- **SNS(유튜브·블로그)**: 각 플랫폼에 들어가 하나하나 확인해야 함
- **포토뉴스**: 홈페이지 게시판에 들어가 개별 글로 진입해야 함

→ "지난 분기 AI 관련 홍보 뭐 했지?" "올해 시상식 사진 어디 있지?" 같은 질문에 답하려면 매번 채널을 돌아다녀야 한다. 신임자 인수인계, 성과 보고, 외부 문의 대응 모두 비효율.

## 2. 목표

**브라우저 한 화면에서 우리 원의 홍보 콘텐츠(최근 5년치)를 카테고리별로 검색·열람할 수 있는 허브**

### 2.1 카테고리 (4종 확정)
- **보도자료**
- **포토뉴스**
- **유튜브**
- **블로그**

(인스타그램은 이번 허브 범위에서 제외 — 5년치 수집 한계 + 활용 비중 고려)

### 2.2 각 항목에 반드시 표시되어야 하는 필드
| 필드 | 설명 |
|---|---|
| **제목** | 콘텐츠 제목 그대로 |
| **게재일자** | YYYY.MM.DD |
| **주요내용** | 한두 줄 요약 |
| **링크** | 원문(또는 영상) URL |

### 2.3 v1 / v2 단계 분리 (사용자 결정)
- **v1**: 카테고리별 리스트가 보이고 각 항목 4필드(제목·게재일자·주요내용·링크)가 노출되는 **기능 우선** 화면
- **v2 이후**: 디자인·시각 형식 업데이트(카드 레이아웃, 썸네일, 차트 등)

→ "일단 작동하는 리스트부터, 모양은 나중에" 노선으로 결정.

## 3. 전체 그림 — 2단계 프로젝트

```
[5년치 원천] ──수집·정리──> [데이터셋 1개] ──시각화──> [브라우저 허브]
보도자료/유튜브/             CSV 또는 시트                v1: 카테고리별 리스트
블로그/포토뉴스             약 1,000~2,000행 예상       v2: 디자인 업그레이드
```

데이터셋 없이 화면부터 만들면 빈 껍데기가 된다. **5년치 데이터 확보가 1단계 핵심.**

## 4. 1단계 — 5년치 데이터 수집

### 4.1 데이터 스키마 (CSV/시트 한 행)

**v1 필수 컬럼** (화면에 그대로 노출)
- `날짜` / `채널` / `제목` / `주요내용` / `링크`

**보조 컬럼** (수집 시 함께 채우되, v2부터 화면 활용)
- `분야태그` / `썸네일URL` / `비고`

분야태그 후보: 연구성과 / 산업·기술이전 / 인사 / 행사·시상 / 기관소개 / 기타
주요내용·분야태그는 **AI로 일괄 생성 → 본인 검수**.

### 4.2 채널별 수집 방식 (URL 확정)

| 채널 | 원천 URL | 방식 | 5년치 가능? |
|---|---|---|---|
| 보도자료 | https://www.kitech.re.kr/pages/61 | categoryId 다중 순회 + 페이지네이션 스크래핑 | 가능 (10년치 약 250건 확인) |
| 보도자료 (2026-04-16 이후) | 담당자가 개별 파일로 전달 | 사용자가 `raw/보도자료-pdf/`에 떨궈주면 처리 | 수동 |
| 포토뉴스 | https://www.kitech.re.kr/pages/60 | 페이지네이션 스크래핑(이미지 URL 포함) | 가능 (1,085건) |
| 유튜브 | https://www.youtube.com/@KITECHYOUTUBE | YouTube Data API로 채널 전체 영상 메타데이터 일괄 수집 | 가능 (API 키 필요) |
| 블로그 | https://blog.naver.com/kitechblog | 네이버 블로그 카테고리·페이지네이션 스크래핑 (RSS는 최근분만) | 가능 |

**보도자료 categoryId (2026-05-07 확인)**
- `50` — 보도자료(2026년~), 약 27건
- `49` — 과거(2016-07-08 ~ 2025-01-23), 약 220건
- `51` — 아카이브 (페이지 메뉴에 노출)
- 스크립트 기본값 `[50, 49, 51]` 전체 순회, id 중복 자동 dedup

### 4.3 채널별 기술 메모

- **KITECH 홈페이지 게시판(보도자료·포토뉴스)**: 1주차에 페이지 구조 분석 — 게시글 목록 페이지의 HTML 구조, 페이지네이션 파라미터, 본문 페이지의 제목·날짜·이미지·본문 셀렉터 확인
- **유튜브**: Google Cloud Console에서 YouTube Data API v3 키 발급(무료 할당량 충분). `channels.list` → `playlistItems.list`로 전체 영상 일괄 추출. 게시일·제목·설명·썸네일·URL·조회수 한 번에 가져옴
- **네이버 블로그**: RSS는 최근 글만 제공 → 5년치는 카테고리별 목록 페이지(`PostList.naver?blogId=kitechblog&...`) 페이지네이션 스크래핑 필요. 본문은 iframe 안에 있어 한 단계 더 들어가야 함
- **보도자료 PDF (4-16 이후분)**: 사용자가 `raw/보도자료-pdf/`에 파일 떨굼 → 파일명에 날짜 포함 권장(예: `2026-04-20_제목요약.pdf`) → 스크립트가 PDF 텍스트 추출 → 데이터셋에 합침

### 4.4 산출물

- `data/허브-데이터셋.csv` (또는 구글시트)
- `raw/` 폴더에 채널별 원본
  - `raw/보도자료-pdf/` ← 담당자 전달분(2026-04-16 이후)
  - `raw/포토뉴스-이미지/`
  - `raw/유튜브-메타/`

## 5. 2단계 — 브라우저 허브 (호스팅 옵션)

### 옵션 S. 스트림릿 (Streamlit) — 유력 후보

- 파이썬 한 파일로 데이터 읽고 카테고리 탭·검색·필터 화면 즉시 구성
- 공유:
  - **Streamlit Community Cloud** (무료, 공개) — 외부 공개 가능 데이터에 한해
  - **사내 서버 배포** — 내부망 전용, 보안 정책 부합
- 5년치 1,000~2,000행 규모는 가볍게 처리
- v1(리스트)에서 v2(카드·차트·검색 에이전트)로 같은 코드베이스에서 점진 확장 용이

### 옵션 A. 구글 워크스페이스(시트 + Looker Studio)
- 데이터: 구글시트, 화면: Looker Studio 대시보드
- 가장 빠르고 운영 쉬움. 단, 외부 SaaS라 보안 검토 필요

### 옵션 B. Notion / Airtable
- DB + 갤러리·캘린더·테이블 뷰
- 외부 SaaS, 보안 검토 필요

### 옵션 C. 내부망 정적 사이트
- 데이터 JSON + 정적 HTML 검색·필터
- 기관 내부 웹서버/그룹웨어에 배포
- 외부 SaaS 불가한 경우 대안

### 5.1 v1 화면 사양 (스트림릿 기준)

```
[ 보도자료 ] [ 포토뉴스 ] [ 유튜브 ] [ 블로그 ]   ← 카테고리 탭
─────────────────────────────────────────────
🔍 검색  📅 기간(연·월)
─────────────────────────────────────────────
2026.05.06 │ ○○ 기술 이전 협약 체결
            민간기업과 신소재 상용화를 위한 기술이전 협약 체결
            → 원문 보기

2026.05.05 │ AI 경진대회 시상식
            우수상 △△팀, 최우수상 □□팀 선정
            → 원문 보기

2026.05.04 │ 산학협력 협약 체결
            ...
─────────────────────────────────────────────
```

- 탭 선택 → 해당 카테고리의 5년치 리스트가 최신순으로 표시
- 4필드(제목·게재일자·주요내용·링크) 명확히 노출
- 검색·기간 필터는 v1 기본 제공
- 디자인·썸네일·카드 레이아웃은 **v2 업데이트**에서

├── scripts/                   ← 수집 스크립트

## 8. 결정 대기 사항

해소된 항목 (2026-05-07): 블로그 플랫폼(네이버), 유튜브 채널(@KITECHYOUTUBE), 보도자료·포토뉴스 URL.

남은 결정 사항:
1. **호스팅 위치** — 스트림릿을 ① Community Cloud(외부, 공개 데이터 한정) ② 사내 서버 ③ 둘 다 가능 중 어디에 배포할지 → IT 정책 확인 필요
2. **분야태그 체계** — 6개 분류(연구성과·산업기술이전·인사·행사시상·기관소개·기타)로 시작할지 조정할지
3. **공개 범위** — 본인 전용 / 홍보실 공유 / 기관 전체 열람
4. **YouTube Data API 키** — Google Cloud Console에서 발급 필요(무료). 유튜브 수집 스크립트 동작에 필수
5. **보도자료 PDF 전달 방식** — 담당자(2026-04-16 이후분) 파일을 `raw/보도자료-pdf/`에 떨궈 둘지, 별도 클라우드 공유로 받을지
6. **(추후) v2 디자인 방향** — 카드 그리드 / 매거진 / 세로 리스트 중 — v1 운영해 보고 결정
